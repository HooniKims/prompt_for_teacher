## What to build

완료된 프롬프트 설계 결과를 새 버전 저장 구조로 보관하고, 현재 앱이 이미 쓰는 기존 저장 데이터를 새 구조로 이관한다.

저장된 세션은 최종 프롬프트, 참고 안내, 초기 요청, 답변, 핵심 메시지를 포함해야 한다. 기존 저장 키에서 읽은 초안도 사용자가 잃지 않도록 새 세션 목록으로 옮겨야 한다.

## Acceptance criteria

- [ ] 완료된 세션은 버전이 붙은 새 저장 키에 저장된다.
- [ ] 세션 목록은 최근 20개까지만 보관한다.
- [ ] 기존 저장 키의 데이터는 새 세션 구조로 이관된다.
- [ ] 이관에 성공하면 기존 데이터가 중복 표시되지 않는다.
- [ ] 이관에 실패하면 기존 데이터는 지워지지 않고 앱은 빈 기본값으로 계속 동작한다.
- [ ] 저장된 세션을 클릭하면 최종 프롬프트와 참고 안내를 다시 볼 수 있다.
- [ ] `node --test`로 저장, 목록 제한, 기존 데이터 이관을 검증할 수 있다.
